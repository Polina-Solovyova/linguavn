# LinguaVN — образовательная платформа изучения английского через визуальные новеллы

Веб-приложение для изучения английского языка на уровнях **B1 / B2 / C1**,
в основе которого лежит жанр визуальной новеллы с разветвлённым сюжетом,
словарём, квизами и системой достижений.

## Функциональность

### Читатель визуальных новелл
- Разветвлённый сюжет с выборами, влияющими на развитие истории
- Несколько концовок в зависимости от принятых решений
- Прогресс сохраняется автоматически (с дебаунсом); можно продолжить с того же места
- Клавиатурная навигация: `Space`/`Enter` — продолжить, цифры — выбрать вариант
- Фоновые изображения, портреты персонажей, фоновая музыка

### Словарь и аннотации
- Ключевые слова прямо в тексте новеллы выделены интерактивными ссылками
- Клик на слово — всплывающая карточка с транскрипцией, переводом, определением и примером
- Из попапа можно сразу добавить слово в свой словарь
- Отдельная страница словаря для каждой новеллы: фильтр по уровню (B1/B2/C1), поиск, статус «учу»

### Квизы
- Тест по завершении главы: вопросы на понимание текста, лексику, грамматику и идиомы
- Форматы: одиночный и множественный выбор
- После отправки — экран разбора: правильные ответы и объяснения к каждому варианту
- История попыток сохраняется в профиле

### Геймификация
- **Очки** начисляются за изученные слова (+2 за слово) и пройденные квизы (+10 × результат)
- **Стрик** — счётчик дней подряд с активностью
- **Достижения** (8 штук): за первую и пятую новеллу, за количество слов (5 / 10 / 50), за квизы

### Личный кабинет
- Вкладка «Прогресс» — список новелл с процентом прохождения
- Вкладка «Достижения» — заработанные и доступные, дата получения
- Вкладка «Квизы» — история попыток с баллами
- Выбор языкового уровня (B1 / B2 / C1), который влияет на сложность текстов внутри новелл
- Статистика: завершено новелл, слов изучено, квизов пройдено, стрик, очки

### Новелла «City Moves: A Fresh Start»
Готовое учебное произведение, включённое в платформу:
- 77 узлов, 6 точек выбора, 3 концовки (великолепная / хорошая / нейтральная)
- Сюжет: стажёр Alex приезжает в Лондон на собеседование в стартап Nexus Tech
- 3 персонажа с портретами, 7 фоновых сцен
- Адаптация текстов под B1 / B2 / C1 в ключевых точках сюжета
- 20 словарных единиц (фразовые глаголы, идиомы, деловая лексика) с транскрипцией и примерами
- 8-вопросный квиз по главе

## Схема базы данных

| Модель            | Назначение                                                  |
|-------------------|-------------------------------------------------------------|
| `UserProfile`     | Расширение User: языковой уровень, очки, стрик, предпочтения |
| `Novel`           | Новелла: метаданные, уровень, жанр, ссылка на JSON-сценарий  |
| `UserProgress`    | Прогресс пользователя по новелле: текущий узел, состояние переменных, процент |
| `VocabularyWord`  | Словарное слово: транскрипция, перевод, определение, пример  |
| `UserVocabulary`  | Слова, добавленные пользователем в личный словарь            |
| `Achievement`     | Определение достижения: код, название, условие, очки         |
| `UserAchievement` | Связь пользователя с полученным достижением + дата           |
| `Quiz`            | Квиз, привязанный к новелле                                  |
| `QuizQuestion`    | Вопрос квиза с типом (single / multiple)                     |
| `QuizChoice`      | Вариант ответа: текст, правильность, объяснение              |
| `QuizAttempt`     | Результат попытки: баллы, ответы пользователя, пройден/нет   |

## API

Все эндпоинты требуют JWT-аутентификации (кроме `auth/*`).
Swagger-документация доступна по адресу `/swagger/` при запущенном сервере.

### Аутентификация
| Метод | URL                   | Описание              |
|-------|-----------------------|-----------------------|
| POST  | `/api/auth/register/` | Регистрация           |
| POST  | `/api/auth/login/`    | Вход, выдача токенов  |
| POST  | `/api/auth/refresh/`  | Обновление access-токена |
| POST  | `/api/auth/logout/`   | Выход                 |

### Профиль
| Метод | URL                       | Описание                     |
|-------|---------------------------|------------------------------|
| GET   | `/api/profile/`           | Полный профиль + прогресс + достижения |
| PATCH | `/api/profile/update/`    | Изменить уровень / предпочтения |
| POST  | `/api/profile/avatar/`    | Загрузить аватар             |
| GET   | `/api/profile/stats/`     | Сводная статистика           |

### Новеллы и прогресс
| Метод | URL                                | Описание                |
|-------|------------------------------------|-------------------------|
| GET   | `/api/novels/`                     | Список (фильтр: level, genre) |
| GET   | `/api/novels/<id>/`                | Карточка новеллы        |
| GET   | `/api/novels/<id>/progress/`       | Прогресс пользователя   |
| PUT   | `/api/novels/<id>/progress/update/`| Сохранить прогресс      |

| DELETE | `/api/vocabulary/<id>/forget/`    | Убрать из «Учу»       |

### Квизы
| Метод | URL                           | Описание                  |
|-------|-------------------------------|---------------------------|

## Формат JSON-сценария

Сценарий хранится как JSON-файл, привязанный к записи новеллы.
Рантайм загружает файл один раз и обходит граф узлов без обращения к БД.

```jsonc
{
  "version": "1.0",
  "meta": { "title": "...", "language": "en", "level": "B2" },
  "assets": {
    "characters": {
      "emma": { "name": "Emma", "image": "https://..." }
    },
    "backgrounds": {
      "office": { "image": "https://..." }
    },
    "music": {}
  },
  "start": "n_intro",
  "nodes": [
    { "id": "n_intro",  "type": "say",    "text": "...", "character": "emma", "background": "office", "next": "n_q1" },
    { "id": "n_q1",     "type": "choice", "prompt": "What do you say?",
      "choices": [
        { "text": "Hello!", "next": "n_hello", "set": { "polite": true } },
        { "text": "...",    "next": "n_ignore" }
      ]
    },
    { "id": "n_check",  "type": "if",     "var": "polite", "equals": true, "then": "n_good", "else": "n_bad" },
    { "id": "n_set",    "type": "set",    "vars": { "key": "value" }, "next": "n_next" },
    { "id": "n_jump",   "type": "jump",   "next": "n_target" },
    { "id": "n_end",    "type": "end" }
  ]
}
```

### Типы узлов

| Тип      | Поля                                       | Поведение рантайма              |
|----------|--------------------------------------------|---------------------------------|
| `say`    | `text`, `character?`, `background?`, `next` | Показывает реплику, ждёт клика  |
| `choice` | `prompt?`, `choices[]`, `background?`      | Показывает варианты, ждёт выбора |
| `set`    | `vars`, `next`                             | Записывает переменные, проходит автоматически |
| `if`     | `var`, `equals`, `then`, `else`            | Ветвление по значению переменной, проходит автоматически |
| `jump`   | `next`                                     | Безусловный переход, проходит автоматически |
| `end`    | —                                          | Завершает новеллу               |

## Адаптация по уровню

Языковой уровень пользователя (B1 / B2 / C1) из профиля инжектируется
в переменную `_level` при старте новеллы. Сценарий использует узлы `if`,
чтобы переключаться между версиями текстов разной сложности:

```
n_level_gate (IF _level == "B1") → n_text_b1
                                 → n_check_c1 (IF _level == "C1") → n_text_c1
                                                                    → n_text_b2
```

Лексика, грамматика и стилистика трёх веток различаются:
- **B1** — простые предложения, базовая лексика, прямая речь
- **B2** — сложные конструкции, фразовые глаголы, идиомы
- **C1** — сложноподчинённые структуры, академическая и деловая лексика

## Запуск локально

### Backend

```bash
cd backend
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
python manage.py migrate
python manage.py seed_novel        # создаёт новеллу, словарь, квиз и достижения
python manage.py createsuperuser   # опционально, для /admin/
python manage.py runserver         # http://localhost:8000
```

Swagger-документация: [http://localhost:8000/swagger/](http://localhost:8000/swagger/)

### Frontend

```bash
cd web
npm install
npm start    # http://localhost:3000
```
